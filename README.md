# site-home-guide

这是一个用于归档和发布多个独立 HTML 页面的仓库。本仓库不针对任何单个域名或具体网站，仅作为静态 HTML 页面的集中存放与管理空间。

## 项目简介

`site-home-guide` 旨在提供一个简洁、通用的 HTML 页面归档方案。你可以将独立的 HTML 文件放置于此，并通过 GitHub Pages 或其他静态托管服务进行发布。每个页面保持独立，互不依赖，便于维护与分享。

## 目录说明

```
site-home-guide/
├── index.html          # 仓库首页（可选）
├── pages/              # 存放归档的 HTML 页面
│   ├── example.html
│   └── ...
├── assets/             # 静态资源（CSS、JS、图片等）
│   ├── css/
│   ├── js/
│   └── images/
├── README.md           # 本文件
└── .gitignore
```

- `pages/`：所有独立 HTML 页面存放于此，文件名建议使用有意义的英文或数字。
- `assets/`：页面引用的公共样式、脚本和图片资源。
- `index.html`：仓库根目录下的默认首页（可选），用于展示页面列表或导航。

## 页面归档说明

- 每个 HTML 文件应自包含，尽量不依赖外部资源（或统一引用 `assets/` 下的资源）。
- 页面之间无耦合，删除或新增某个页面不会影响其他页面。
- 建议在页面内添加合适的 `<title>` 和 `<meta>` 描述，以便于检索和预览。

## 维护说明

- 添加新页面：在 `pages/` 目录下创建 HTML 文件，并确保资源路径正确。
- 更新页面：直接修改对应 HTML 文件，注意保持版本历史。
- 发布：启用 GitHub Pages 后，仓库内容将自动部署为静态站点。访问路径为 `https://<用户名>.github.io/site-home-guide/`（请替换为实际用户名）。

## 注意事项

- 本仓库不存储任何敏感信息或受版权保护的内容。
- 请勿放置任何违反平台使用条款的页面。
- 提交前建议使用 HTML 验证工具检查页面格式。

## 贡献

欢迎通过 Issues 或 Pull Requests 提出改进建议。请确保修改内容符合本仓库的通用定位。
